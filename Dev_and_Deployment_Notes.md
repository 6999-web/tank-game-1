# 🛡️ Web 坦克大战：从零开发到云端全线调优部署实录

本项目是一个基于 **HTML5 Canvas + Phaser.js** 引擎开发的经典坦克大战小游戏。本文档详细记录了项目的开发流程、遇到的核心技术难题（502/404 错误）及其最终的工业级解决方案。

---

## 一、 开发历程：零资源、全动态渲染

### 1. 技术选型
*   **引擎**：Phaser.js v3 (Arcade Physics)。
*   **构建工具**：Vite (提供极速的开发体验和高效的静态打包)。
*   **架构设计**：采用模块化 Class 设计，分离了坦克逻辑 (`Player/Enemy`)、子弹系统、地图/关卡系统以及碰撞检测系统。

### 2. 核心创新：自研纹理生成器 (`generateTextures`)
为了实现“直接运行、无需外部素材”的要求，我在 `MenuScene.js` 中编写了一个动态纹理生成系统：
*   **原理**：利用 Phaser 的 `Graphics` API 在内存中绘制坦克、墙体、水面和草丛的像素图形。
*   **优势**：无需加载任何图片文件，彻底消除了图片加载失败导致的黑屏问题，且减少了磁盘 IO。

## 二、 部署实战：解决 502/404 核心顽疾

在将项目部署到腾讯云轻量服务器（Lighthouse）时，我们经历了一系列从“能跑”到“稳定”的调优过程。

### 1. 为什么会出现 502 Bad Gateway？ (重点分析)
**原因 A：端口代理冲突**
*   **现象**：最初尝试使用 38888 端口。
*   **诊断**：该服务器之前运行过 Nginx 反向代理。新启动的文件服务器在绑定 38888 时被 Nginx 的残留配置拦截，网页请求被错误地引向了一个不存在的旧代理路径。
*   **解决**：彻底废弃旧端口，改用一个全新的、纯洁的偏僻端口 **`33333`**（或 `40001` 以上）。

**原因 B：Python 内置服务器的并发瓶颈**
*   **现象**：HTML 能加载，但 CSS/JS 资源报错 502。
*   **原理**：`python3 -m http.server` 是单线程的。当现代浏览器同时发起对 `main.js`、`style.css`、`favicon.ico` 等 5-10 个并发请求时，Python 服务器会排队处理，导致后续请求被连接重置（Reset）。
*   **解决**：放弃 Python，改用基于 Node.js 事件驱动、异步并发的 **`http-server`** (或 `serve`) 工具。

### 2. 为什么会出现 404 Not Found？
*   **诊断**：Vite 打包后的资源路径包含随机哈希值（如 `index-CZf7sdCN.js`）。如果服务器根目录设置不正确，或者在解压 `dist.zip` 时路径嵌套（多了一层 `dist/` 文件夹），就会导致文件系统找不到对应的 JS 文件。
*   **解决**：
    1.  确认 `dist.zip` 内容直接包含 `index.html`。
    2.  启动服务时明确指定目录：`http-server -p 33333 ./dist`。

## 三、 移动端访问深度兼容方案

手机端访问云端高位端口（如 33333）往往伴随以下障碍：

1.  **运营商拦截**：移动卡/联通卡有时会屏蔽非标准 80/443 的高位端口。
    *   **对策**：换用更宽泛的端口段（如 30000 左右），或在必要时切回手机流量/更换 WiFi 测试。
2.  **HTTPS 强制跳转劫持**：手机浏览器倾向于访问 `https://`。
    *   **对策**：必须手动且完整地输入 **`http://101.33.210.169:33333`**。
3.  **虚拟摇杆适配**：
    *   **开发方案**：引入了双指针触控监听。左侧显示半透明 Joystick，右侧显示红色按钮。
    *   **优化**：在 CSS 中加入 `touch-action: none`，防止手机端玩游戏时页面发生缩放或上下滚动。

## 四、 最终产出的工业级部署 Skill

### 一键启动脚本（服务器执行）
为了确保持久运行且稳定可靠，我们最终执行了以下“金牌操作”：
```bash
# 1. 杀掉所有单线程/有问题的服务
pkill -f python ; pkill -f http-server

# 2. 进入带 index.html 的目录并启动高吞吐服务器
cd /root/dist
nohup npx http-server -p 33333 > ~/game_access.log 2>&1 &
```

### 访问验证
- **PC测试**：`curl -I localhost:33333` → 返回 `200 OK`
- **公网测试**：`curl.exe -I 101.33.210.169:33333` → 返回 `200 OK`
